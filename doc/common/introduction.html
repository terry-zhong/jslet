Jslet Architecture


  1. UI Layer:
    Jslet UI control is standard html tag, only one customized attribute: data-jslet is added in it, so jslet is non-invasive.
    The customized attribute value is a json string, example: 
    
<input data-jslet='{type:"DBText", dataset:"employee", field:"name"}' type="text" />
  2. Data Layer:
    Jslet data layer is non-invasive too, Jslet data layer does not do any additional processing to the data comes from server.
  3. Server
    Jslet data layer(jslet.data.DataProvicer) exchanges data with server, the default format of data is JSON.

Jslet Modular Loading

Jslet framework provides a powerful loader, it can load js file, css file, even your js file.

Jslet loader js file is jslet.loader.js, you can find it in jslet project folder.

Jslet module Configuration

Jslet module Configuration is in the begin of jslet.loader.js

Form:
jslet._initialModules = [
{ name: "jquery", src: "lib/jQuery.js", baseOnLoader: true },
{ name: "base", src: "base/jslet.base.js", baseOnLoader: true },
{ name: "dataset", src: "data/jslet.dataset.js", deps: "jquery,base,provider", baseOnLoader: true },
......

Comments：
  1. name: module name;
  2. src: the js file url of module
  3. deps: dependency modules, separated with comma(,)；
  4. baseOnLoader: optional value: true or false, true - the module js file is relative to jslet.loader.js, false - otherwise.
