jslet.data.

Class Dataset


  • jslet.data.Dataset is the core class of Jslet framework. Features:
    1. Full support local data manipulating: add, modify, delete(Demo), sort data(Demo), filter data(Demo), find data(Demo)
    2. Support display format;
    3. Support edit mask(Demo)
    4. Multiple level validation, prevent any error data(Demo)
    5. Support formula field, you can use it in dynamical calulations(Demo)
    6. Support FieldLookup, FieldLookup is the relation of datasets, many UI controls use field lookup to construct UI.(Demo)
    7. Support GroupField, you can use group field to sort data(Demo)
    8. Support DatasetField, you can use dataset field to display Master-Detail style data easily(Demo)
    9. Support FieldChain, you can use field chain to get, sort, filter and find data.(Demo)
    10. Support Context Rule(Demo)
    11. Synchronize auto-increase field value from server automatically;
    12. Can clone dataset structure(Demo)
    13. Can export data(Demo)
    Since:
    1.0
    See Also:
    Field, DataProvider, ContextRule, Jslet Expression Synctax, FieldLookup, DatasetEvent, DatasetStatus, RefreshEvent

        • submit

          submit(url)

          Send changed data to server.
          If server side save data successfully and return the changed data, Jslet can refresh the local data automatically.

          Cause key value is probably generated at server side(like sequence), we need an extra field which store an unique value to update the key value, this extra field is named '_s_', its value will start a letter 'i', 'u' or 'd', and follow a sequence number, like: i1, i2, u1, u2, d1, d3,.... You don't care about it in client side, it is generated by Jslet automatically.

          At server side, you can use the leading letter to distinguish which action will be sent to DB('i' for insert, 'u' for update and 'd' for delete) If the records need be changed in server(like sequence key or other calculated fields), you should return them back.Notice: you need not to change this value of extra field: '_s_', just return it. Example: 

          
dsFoo.insertRecord();
dsFoo.setFieldValue('name','Bob');
dsFoo.setFieldValue('code','A01');

dsFoo.confirm();
dsFoo.submitUrl('../foo_save.do');
dsFoo.submit();
          Since:
          1.0

        • submitSelected

          submitSelected(url, deleteOnSuccess)
          Send selected data to server whether or not the records have been changed.
          Under some special scenarios, we need send user selected record to server to process. Sever side need not send back the processed records. Example: 
          
 //Audit the selected records, if successful, delete the selected records.
 dsFoo.submitSelected('../foo_audit.do', true);
          Parameters:
          url - (Required) String, url.
          deleteOnSuccess - (optional) Boolean, if processing successfully at server side, delete the selected record or not.
          Since:
          1.0
